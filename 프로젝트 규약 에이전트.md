# 프로젝트 규약 에이전트

이 문서는 MwoLab 프로젝트를 수정할 때 지켜야 할 목적, UI 처리 규칙, 탭별 표시 항목, 슬롯 표시 규격을 정리한다.

## 1. 프로젝트의 목적

- 이 프로젝트는 로컬 MechWarrior Online 게임 데이터에서 추출한 JSON을 기반으로 동작하는 정적 웹 멕랩이다.
- 앱은 `public` 폴더만으로 배포 가능한 브라우저 앱이어야 하며, 멕/장비/로드아웃/옴니포드 데이터는 `public/data`의 JSON에서 읽는다.
- 서버 API, 계정 기능, 온라인 빌드 공유, 위키, 드랍십 기능은 범위 밖으로 둔다.
- 데이터 갱신은 `tools/extract_mwo_data.py`로 수행하며, 앱 코드는 추출 데이터의 구조를 존중한다.
- 사용자는 멕을 빠르게 찾고, 내구도/기동성/쿼크/하드포인트를 확인하고, 여러 멕을 비교할 수 있어야 한다.

## 2. UI 처리에 관한 룰

### 공통 원칙

- UI는 조용하고 밀도 높은 도구형 화면을 유지한다. 장식보다 스캔, 비교, 반복 사용을 우선한다.
- 사용자에게 보이는 주요 텍스트는 한국어를 기본으로 한다. 단, 게임 원본명, 장비명, 스탯 키, 진영명처럼 원 데이터가 영어인 값은 그대로 사용할 수 있다.
- 모든 탭은 같은 선택 상태를 공유한다. 선택 멕은 `state.selectedMech`, 현재 빌드는 `state.currentBuild`, 비교 선택은 `state.compareMechIds`를 기준으로 처리한다.
- 계산값은 하드코딩하지 말고 `mechs`, `equipment`, `loadouts`, `omnipods`에서 파생한다.
- 쿼크 적용 여부는 정보/비교/통계에서 같은 기준을 쓰며, 표시값이 기본값과 달라질 때는 강조 표시를 유지한다.
- 로컬 미리보기 서버는 사용자가 명시적으로 요청하지 않는 한 시작, 점검, 탐색, 검증하지 않는다.

### 전체 레이아웃

- 상단 바는 프로젝트명, 데이터 상태, 요약 스트립, 후원 링크를 표시한다.
- 요약 스트립은 멕랩 빌드 상태 요약용이며, 현재 멕랩 탭이 아닐 때는 숨긴다.
- 왼쪽 멕 브라우저는 정보/비교/현재 멕랩 계열 작업에서 사용하고, 통계 탭에서는 숨긴다.
- 멕 브라우저에는 검색, 리스트 크기 토글, 진영 필터, 체급 필터, 정렬 선택을 둔다.
- 멕 리스트는 체급, 진영, 섀시, 배리언트 순서로 그룹화한다.

## 3. 각 탭 표시 규격

### 현재 멕랩 탭

- 현재 구현에서는 탭 버튼이 비활성화되어 있고, `mechlab` 요청은 정보 탭으로 우회된다.
- 다시 활성화할 경우 다음 영역을 유지한다.
- 배리언트 패널: 멕 이름, 진영, 체급, 톤수, 엔진 범위, Stock/Save/Clear 버튼.
- 쿼크 패널: 적용된 쿼크 목록과 활성 쿼크 수.
- 컴포넌트 패널: 부위별 장갑, 사용 슬롯, 슬롯 한도, 하드포인트, 경고, 장착 아이템.
- 장비 패널: 장비 검색, 장비 패밀리 필터, 선택 장비 요약, 장비 리스트.
- 요약 스트립: Tons, Alpha, Heat, Ammo, Engine, Status.

### 정보 탭

- 기본 활성 탭이다.
- 헤더에는 선택 멕 이름과 `진영 - 체급 - 톤수`를 표시한다.
- 쿼크 적용 토글을 제공한다.
- 정보 카드는 다음 순서로 표시한다.
- 아머 정보: 최대 아머 포인트 총합, 머리, 몸통, 왼쪽 어깨, 오른쪽 어깨, 왼쪽 팔, 오른쪽 팔, 왼쪽 다리, 오른쪽 다리.
- 스트럭쳐 정보: 스트럭쳐 총합과 동일 부위별 수치.
- 종합 내구: 아머 + 스트럭쳐 총합과 동일 부위별 수치.
- 엔진: 최소 엔진, 최대 엔진.
- 기동성: 최대 속도, 가속도, 감속도, 선회 속도, 회전각 X, 회전각 Y, 몸통 회전속도.
- 쿼크 리스트: 쿼크명, 출처, 값.

### 비교하기 탭

- 비교 탭 진입 시 비교 모드가 켜지고 기존 비교 선택과 기준 멕은 초기화한다.
- 왼쪽 멕 리스트에서 최대 15개까지 비교 대상을 선택한다.
- 헤더에는 선택 개수와 선택된 배리언트 코드를 표시한다.
- 컨트롤은 비교 리스트 모두 지우기, 차이 표시, 쿼크 적용을 제공한다.
- 각 멕 열에는 기준 라디오, 배리언트 코드, 제거 버튼, 진영/톤수를 표시한다.
- 비교 그룹은 기본 정보, 내구도 요약, 기동성, 종합 내구, 아머 정보, 스트럭쳐 정보, 쿼크 순서로 표시한다.
- 기본 접힘 그룹은 종합 내구, 아머 정보, 스트럭쳐 정보다.
- 차이 표시가 켜져 있으면 수치 셀에 상승/하락 표시와 색상 강조를 적용한다.
- 기준 멕이 지정되면 모든 수치 차이는 기준 멕 대비로 계산한다.
- 비교 테이블 헤더가 스크롤 밖으로 나가면 고정 오버레이 헤더를 표시한다.

### 통계 탭

- 통계 탭에서는 멕 브라우저를 숨기고 전체 폭 리스트를 사용한다.
- 현재 통계 종류는 내구도다.
- 종합 내구도 총합 기준으로 내림차순 정렬한다.
- 각 행은 순위, 멕 이름, 진영/톤수, 내구도 값, 체급 배지, 하드포인트 배지를 표시한다.
- 표시할 멕이 없으면 빈 상태 문구를 보여준다.

## 4. 슬롯 표시 규격

### 컴포넌트 순서

컴포넌트 표시는 다음 순서를 고정으로 사용한다.

1. Head
2. Left Arm
3. Left Torso
4. Center Torso
5. Right Torso
6. Right Arm
7. Left Leg
8. Right Leg

후방 장갑 컴포넌트(`left_torso_rear`, `centre_torso_rear`, `right_torso_rear`)는 슬롯 카드로 직접 표시하지 않고 아머 계산에만 반영한다.

### 컴포넌트 카드 항목

- 제목: 컴포넌트 표시명.
- 메타: `Armor {armor} · Slots {used}/{limit}` 형식.
- 하드포인트 배지: `{hardpoint_type} {Slots}` 형식.
- 경고: 슬롯 초과, 하드포인트 없음, 누락 아이템 등을 표시.
- Add 버튼: 선택된 장비를 해당 컴포넌트에 추가한다.
- 아이템 영역: 장착 아이템이 있으면 슬롯 아이템 목록, 없으면 `Empty`.

### 슬롯 계산

- 사용 슬롯은 컴포넌트 내장 장비 수와 장착 아이템 슬롯의 합이다.
- 내장 장비 수는 `component.internals.length`로 계산한다.
- 아이템 슬롯은 `item.stats.slots`를 사용한다.
- 슬롯 한도는 `component.slots`를 사용한다.
- 슬롯 한도가 있고 사용 슬롯이 한도를 넘으면 `Slots {used}/{limit}` 경고를 표시한다.

### 하드포인트 계산

- 유효 컴포넌트 정의는 옴니포드 하드포인트를 최우선으로 사용한다.
- 옴니포드 하드포인트가 없으면 기본 컴포넌트 하드포인트를 사용한다.
- 기본 하드포인트도 없으면 로드아웃 아이템에서 무기 하드포인트를 파생한다.
- 멕 리스트용 하드포인트 요약은 `ballistic`, `energy`, `missile`, `ams`, `ecm`만 배지로 표시한다.
- 무기가 장착된 타입의 하드포인트 용량이 0이면 `No {type} hardpoint` 경고를 표시한다.

### 슬롯 아이템 표시 항목

- 아이콘 배지: `item.item_type`의 첫 글자.
- 이름: `item.display_name`.
- 무게 배지: `{tons}t`.
- 슬롯 배지: `{slots} slots`.
- 하드포인트 배지: 무기류처럼 `item.hardpoint_type`이 있을 때만 표시.
- 제거 버튼: `Remove`.
- 아이템 데이터가 없으면 `Missing {item_id}`와 제거 버튼을 표시한다.

### 장비 리스트 표시 항목

- 아이콘 배지: `item.item_type`의 첫 글자.
- 이름: `item.display_name`.
- 우측 값: 장비 무게.
- 배지: `family`, `slots`, 선택적 `hardpoint_type`.
- 필터 패밀리는 All equipment, Weapons, Ammo, Engines, Equipment, Jump jets, MASC, Weapon mods를 유지한다.

## 5. 수치와 표시값 규격

- 장비 무게는 `item.stats.tons`를 우선하고 없으면 `item.stats.weight`를 사용한다.
- 열량은 `item.stats.heat`를 사용한다.
- 알파 데미지는 무기 `damage * numFiring`으로 계산한다.
- 탄약은 ammo 아이템의 `numShots`를 합산한다.
- 엔진은 장착 엔진의 `stats.rating`을 사용하며, 멕의 최소/최대 엔진 범위를 벗어나면 경고한다.
- 총 톤수는 기본 톤수, 장비 톤수, 아머 톤수를 합산한다.
- 아머 톤수 계산의 기본 armor per ton은 업그레이드 데이터가 없으면 32를 사용한다.
- 표시 숫자는 불필요한 소수점을 줄이고, 비교 테이블에서는 좁은 셀에 맞도록 자릿수를 줄인다.

## 6. 변경 체크리스트

- 새 UI 항목을 추가할 때는 정보 탭, 비교 탭, 통계 탭 중 어디까지 반영할지 먼저 정한다.
- 멕 리스트 카드에 수치를 추가하면 쿼크 적용 여부와 기본값 대비 강조 상태를 함께 검토한다.
- 슬롯/하드포인트 계산을 바꿀 때는 멕랩 컴포넌트 카드, 멕 리스트 하드포인트 배지, 통계 행 표시가 같은 기준을 쓰는지 확인한다.
- 데이터 구조를 바꿀 때는 추출 스크립트와 `public/data/index.json`의 파일 연결을 함께 확인한다.
- 기존 작업자의 수정이 있는 파일은 되돌리지 말고 필요한 부분만 좁게 수정한다.
